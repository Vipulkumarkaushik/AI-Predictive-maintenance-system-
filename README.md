Predictive Maintenance & Thermal Anomaly Forecasting PipelineIndustrial IoT Big Data FrameworkProject ObjectiveThis system upgrades traditional threshold-based hardware monitoring (ATmega328/LM35) into a distributed, predictive AI pipeline. The goal is to identify mechanical degradation and imminent thermal failure using high-velocity sensor telemetry before physical damage occurs.System ArchitectureThe project is divided into four distinct layers to ensure scalability and statistical rigor:Edge Intelligence Layer (TinyML Simulation): Generates high-frequency synthetic sensor data. Beyond raw temperature, it computes Thermal Gradients ($\Delta T / \Delta t$) and rolling averages at the source to reduce cloud bandwidth.Statistical Profiling (R): Utilizes R-based Exploratory Data Analysis (EDA). Boxplots and Density Plots were used to mathematically define the interquartile range (IQR) for normal operation versus critical outliers.Distributed ML Engine (Apache Spark): Implements a Random Forest Classifier via PySpark. Distributed processing allows the system to scale to millions of rows, achieving an accuracy of 99.96% by analyzing multi-dimensional feature sets.Operations Dashboard (PowerBI & Web): A dual-interface deployment. PowerBI provides executive-level KPI tracking, while a web-based UI provides real-time inference logs for on-site engineers.Technical SpecificationsData Volume: 50,000+ telemetry records.Core Model: Random Forest (Spark MLlib).Primary Features: Temperature_C, Rolling_Avg_5m, Temp_Gradient.Inference Result: Binary Classification (0: Normal, 1: Anomaly).
